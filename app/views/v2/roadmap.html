Roadmap

This is what we plan to deliver in the coming months, and what we expect to work on over the longer term.

It’s a guide that’s updated quarterly, and things might change. Get in touch with the team, via gov-uk-paas-support@digital.cabinet-office.gov.uk, to find out more about our roadmap and to let us know about features you need.

January to March 2018

  • Backing services (public beta): make Redis, Elasticsearch and MongoDB available for production use
  • UK hosting (alpha): create a UK-based GOV.UK PaaS instance and scope the work involved in making this available for production use
  • internal-only applications (alpha): explore ways of enabling private, non-internet-facing communication between applications, including service discovery
  • GOV.UK PaaS web admin tool (alpha): let users see the platform resources they’re using and manage basic admin tasks themselves
  • publish a pricing calculator, so users know how much GOV.UK PaaS will cost them
  • improve the support we provide for Docker images

April 2018 and onwards

  • UK-based GOV.UK PaaS (private beta)
  • automated billing
  • enhanced protective monitoring
  • file storage
  • metrics for backing services
  • improved access control (IP whitelisting/auth)
  • RabbitMQ
  • Logging as a Service

Longer term possibilities

  • PSN connectivity
  • CDN as a Service
  • CI as a Service
  • Memcache
  • VM isolation

Development stages

Most of the features we’re building go through the following stages of development:

  • alpha: exploring the problem space and investigating possible solutions
  • private beta: an early working version of the feature that’s tested by selected partners in a development environment - not suitable for use in production
  • public beta: a production-ready version of the feature that has been subjected to Cabinet Office/GDS information approval and security governance
  • live: a production-ready service that has been improved based on feedback during public beta
